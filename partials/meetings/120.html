O prezentacji:

Programowanie sieciowe w Javie od początku było jej kluczowym elementem. Gdy jest konieczna komunikacja między systemami to rozwiązania takie jak REST, SOAP czy JMS najczęściej pasują idealnie. Są jednak problemy, gdzie ten poziom rozwiazan nie jest odpowiedni, np.:

  • integracje istniejacych systemow, stworzonych w roznych technologiach, czesto przed era RESTa, Javy
  • specyficzne protokoly dla danej domeny systemu
  • rozwiązania infrastruktury, programowanie systemowe

W takich sytuacjach konieczne jest użycie narzędzi i API niższego poziomu - gniazdek. I choć również tutaj Java dostarcza dobrego wsparcia to jednak głębsza wiedza jest potrzebna, aby tworzone rozwiazanie było poprawne, bez ukrytych problemów, o które dość łatwo.

Celem prezentacji jest pokazanie możliwości Javy w programowaniu sieciowym na poziomie gniazdek poprzez omówienie:

  • istotnych API i mozliwych opcji
  • kluczowych elementow protokolu TCP i wynikajacych zagrozen
  • przykladu implementacji wlasnego protokolu warstwy aplikacji
  • roznych modeli obslugi IO: blokujacy, nieblokujacy, asynchroniczny
  • wybranych szczegolow implementacyjnych na poziomie systemu operacyjnego
  • ograniczen Javy w programowaniu systemowym
  • propozycji architektury
